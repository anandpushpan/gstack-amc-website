## 🔴 Must-fix (bugs, accuracy, accessibility)

| # | Issue | Fix | Status |
|---|-------|-----|--------|
| A1 | **Mobile nav broken.** Below 900px the nav is `display:none` and the hamburger isn't wired. Mobile visitors cannot navigate. Most UK car buyers are on phones. | Wire the toggle to a dropdown/overlay menu. | ✅ |
| A2 | **Dealer margin example undersells the pitch.** £6,900 buy − £1,759 recon → £9,095 retail = £436 (~6%). No dealer is impressed by 6%, and the numbers don't reconcile with the moat card's same car (£3,950 market). | Use a realistic healthy-margin example and clearly make it a different car than the moat card. | ✅ |
| A3 | **"Results in seconds" overpromises.** The AI visual assessment analyses photos; it isn't instant. Wrong place to round up for a trust product. | "Data checks in seconds" or remove. | ✅ |
| A4 | **"Over 1,000 data points" unverified** (carried from old marketing). | Confirm it's real or soften the claim. | ✅ |
| A5 | **Low-contrast text.** `.valnote` (`--fg3` on `--bg2`) ≈ 2.5:1, fails WCAG AA. | Darken to `--fg2`. Re-check `.feat .cr` and `.reg-meta`. | ✅ |
| A6 | **No visible focus states** on links/buttons; keyboard users can't see focus. | Add a `:focus-visible` outline using `--focus-ring`. | ✅ |
| A7 | **Footer social "icons" aren't real links** (`<span>` + title only) — not focusable, not labelled. | Make them `<a>` with `aria-label`, or remove until handles are unified. | ✅ |

## 🟡 Should-improve (conversion, content)

| # | Issue | Fix | Status |
|---|-------|-----|--------|
| B1 | **The page never shows the product.** The actual report (build sheet, AI repair breakdown, valuation trend) is the strongest asset and is never shown. Likely the biggest conversion lever. | Add a report preview image/mockup near the hero or features. | ✅ |
| B2 | **Two near-identical valuation cards** (moat + dealer) feel repetitive. | Vary one layout. | ✅ |
| B3 | **Credibility gap for a trust product.** Fake reviews correctly removed, but no third-party trust markers remain and the dashed "newly launched" box reads weak mid-page. | Add legitimate markers: company reg number, payment-provider logos (if used), named data sources. Consider moving the "new" note lower. | ✅ |
| B4 | **Free-report signup friction not set.** If the free report needs an account (live site does), the CTA doesn't say so → surprise wall. | State it near the CTA ("free, quick account, no card"). | ✅ |
| B5 | **Dealer section mid-B2C page** is a context switch. | Acceptable for "serve both"; consider a clearer divider or a dedicated `/for-dealers` page. | ⏭️ |

## 🟢 Nice-to-have (polish, performance, SEO)

| # | Issue | Fix | Status |
|---|-------|-----|--------|
| C1 | **No structured data.** (Old fabricated `AggregateRating` correctly removed.) | Re-add HONEST `Organization` + `FAQPage` (matching the visible FAQ). Never re-add `AggregateRating` until reviews are real. | ✅ |
| C2 | **lucide pinned to `@latest`** — a CDN change could break icons in production. | Pin a specific version. | ✅ |
| C3 | **~12 font files** (Onest ×4, Manrope ×5, Plex Mono ×3) slow first load. | Drop unused weights. | ✅ |
| C4 | **No favicon / OG image.** | Add favicon + 1200×630 OG image. | ✅ |
| C5 | **No `prefers-reduced-motion`** handling for hover transforms. | Add a reduced-motion media query. | ✅ |
| C6 | **Heading hierarchy** skips `h2`→`h4` on features/steps. | Use `h3`. | ✅ |

## Strategic note
The page is **B2C-led with a dealer section**, consistent with the owner's "serve both" answer.
Our research (`../../g-stack-research/`) repeatedly points at **trade-first** as the sharper wedge.
A **dealer-led hero variant** could be produced to A/B against this one. ⏭️
