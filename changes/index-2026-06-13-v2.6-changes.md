# v2.6 Changes: Redesign the module-card grid ("Everything in one valuation report")

**Date**: 2026-06-13

## The problem

The six module cards under "Everything in one valuation report" each used a real
report screenshot as a full-bleed top image, with the credit-cost pill positioned
absolutely over the top-right corner. Two issues made the section read as broken:

1. The report screenshots are tall (e.g. 1015x460). Cropping them into a 16:9
   box with `object-fit:cover` showed only a thin top sliver of each section,
   not a usable preview.
2. The dark credit pill floated on top of that sliver, colliding with whatever
   was behind it. Before the image painted, the alt text showed through behind
   the pill, which looked like a layout fault.

## The fix (new layout)

Replaced the screenshot-led cards with clean icon + credit-chip cards:

- A top row holds a tinted, rounded icon tile on the left and the credit-cost
  chip on the right. The chip is no longer overlaid on anything, so it always
  reads cleanly.
- Title and description sit below in a simple vertical stack.
- The paid AI module keeps its purple "Paid add-on" pill and now gets a purple
  icon tile and a purple credit chip to mark it as the upsell.

The real report screenshots are not lost: the "What's inside every report"
deep-dive section directly below this grid still shows them at full size, where
tall images display properly.

## Verification

- All six cards render; all six lucide icons paint as SVG.
- No horizontal overflow at 390px (document width equals the 390px viewport).
- Desktop (3-up) and mobile (1-up) both clean, no badge/image collision.
- 0 em/en dashes.
