# v2.4 Changes: Fix edge-to-edge hero on mobile

**Date**: 2026-06-13

## The bug

On mobile the hero lookup card and the source-trust strip ran flush to the screen
edges, with no side padding. Two causes:

1. The hero container is `class="wrap hero-in"`. `.wrap` sets `padding:0 24px`,
   but `.hero-in` re-declared the `padding` shorthand (`64px 0 76px`), which reset
   the horizontal padding to 0. Same issue on `.srcstrip .row`.
2. The base `.hero-in` sets `align-items:center` for the desktop grid. When the
   hero switches to `flex-direction:column` on mobile, that `center` stopped the
   card stretching to the container width, so it sized to its content (~394px)
   and overflowed past the screen edges.

## The fix

- `.hero-in` and `.srcstrip .row` now set vertical padding via `padding-top` /
  `padding-bottom` only, so `.wrap`'s 24px horizontal padding is preserved.
- The mobile hero flex sets `align-items:stretch` (plus `min-width:0` on the card
  and hero text) so the card fills the padded content area.

## Result
- Hero card and content now sit 24px from each edge at all mobile widths.
- No horizontal overflow (document width equals viewport at 390px and 430px).
- Desktop unchanged. 0 em/en dashes.
