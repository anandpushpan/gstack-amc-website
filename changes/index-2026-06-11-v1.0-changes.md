## What v1.0 is
A self-contained homepage draft built on the exported AnalyzeMyCar design system
(Onest / Manrope / IBM Plex Mono; blue-teal-lime brand gradient). It positions the product
honestly as a **valuation + condition + OEM-spec** tool, leads with the refurb-adjusted-valuation
differentiator, features a dealer/trade section, and carries zero fabricated trust claims.

## Development history that produced v1.0

### Stage A — initial build (from the strategy + design system)
- Built on the exported design system tokens and component patterns.
- Hero leads with "Know what a used car is really worth" (the valuation moat), not the old
  "Unlock Your Vehicle's True Story."
- Added a dedicated **differentiator section** (refurb-adjusted valuation: £3,950 − £1,759 = £2,191).
- Added a dedicated **dealer/trade band** (the trade wedge).
- Added an **honest trust section** (official data, UK company, "newly launched, real reviews only").
- **Omitted** finance / write-off / stolen claims (sample report doesn't show them; team unsure).
- Featured the **free first report** as the hook (owner-confirmed real).

### Stage B — owner feedback round (Presentation1.pdf) → frozen as v1.0
| # | Feedback | Change made |

| 3 | "Secure checkout" repeated twice in one fold | Removed from hero trust row (kept in source strip) → hero now reads **"Results in seconds"** |
| 4 | Remove em dashes everywhere | All em/en dashes removed site-wide (incl. comments); negatives use a true minus sign |
| 5 | No gradients in number boxes / small spaces | Valuation card header → solid navy; step number badges → solid blue |

| 7 | One primary gradient per fold | Dealer band + CTA band keep full-bleed gradient background; buttons inside are now SOLID |
| 8 | Green line on the dealer CTA | Root cause: gradient button on gradient background. Fixed by making the button solid |
