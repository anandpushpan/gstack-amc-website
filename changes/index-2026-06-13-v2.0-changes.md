# v2.0 Changes: Mobile-first overhaul

**Date**: 2026-06-13

## Why this is a major version

The core user is a buyer standing at a dealer, on their phone, who wants to check
a specific car's real value to push back on a quoted price, fast, before the
moment passes. The page was a desktop-style marketing layout that merely stacked
on mobile. v2.0 restructures it around that forecourt moment. Desktop is unchanged
(every change is gated behind the mobile breakpoint).

## What changed (7 phases)

### 1. Action-first hero (mobile)
On phones the reg/VIN lookup card now comes FIRST, above the headline and pitch.
A forecourt user sees the plate input the instant the page loads instead of
scrolling past marketing copy. The headline and value story sit below. Desktop
keeps the copy-left / card-right two-column layout.

### 2. One-tap free report
On mobile the four module rows collapse behind a "Customize what's included"
toggle. The default path is reg -> "Get my first report free", no toggling. The
summary ("Your first report: free, 3 modules") stays visible so the user still
knows what they get. Tapping Customize reveals the full picker. Desktop keeps the
picker open (it's the browser's hook).

### 3. Sticky thumb-zone CTA (mobile)
A "Check a car free" bar pins to the bottom of the screen once the hero scrolls
out of view, so the action is always one thumb-tap away. Respects notch safe-area
insets; hidden while the hero input is on screen.

### 4. Input ergonomics & touch targets
Reg and VIN inputs get `autocapitalize=characters`, `autocorrect=off`,
`spellcheck=false`, `inputmode`, and `enterkeyhint=go` so the right keyboard
appears on first tap. Footer social links bumped 34px -> 44px; reg/VIN toggle
buttons enlarged on mobile. All interactive targets meet the 44px minimum.

### 5. Mobile section order
The dealer/trade band ("Know your margin before you buy the stock") now sits
below the consumer flow on mobile instead of interrupting it mid-page. The body
is a flex column; the dealer band and footer get explicit order on mobile only.
Desktop source order is unchanged.

### 6. Performance for 4G
Logo gets `fetchpriority=high`; the icon library loads `defer` and initialises on
DOMContentLoaded so it never blocks first paint. Report screenshots stay lazy;
fonts stay preconnected. The first screen paints fast on dealer-grade signal.

### 7. Honesty copy fix
Hero badge changed from "first full report is FREE" to "first report is FREE",
matching the real model (you assemble the report; AI Visual is a paid add-on).

## Verification
- Desktop confirmed unchanged: picker visible, no customize toggle, no sticky bar,
  no trust cue, dealer band in its original mid-page position.
- Mobile (390px): input card first, picker collapsed, customize toggle opens rows,
  sticky bar appears on scroll, dealer band pushed below the final CTA, trust cue
  visible, no horizontal overflow.
- Interactions tested: customize disclosure, sticky-bar show-on-scroll, module
  toggle summary.
- 0 em/en dashes.

## Credit model (unchanged, owner-confirmed)
Basic Info 1cr, Build Sheet + Valuation 2cr, DSB + MOT 3cr, AI Visual 4cr; full
report 10cr. First report free for all except AI Visual, a paid add-on even on
the first report.
