# Changes — index.html v1.3

**Date:** 2026-06-11  ·  **Snapshot:** `versions/index-2026-06-11-v1.3.html`

## What changed

### 1. Feature cards are now screenshot-led
The "Everything in one valuation report" section was a grid of icon + text cards. Each card now
**leads with a real screenshot of that section from a sample report**, with the credit cost shown
as a badge on the image, so a visitor can see exactly what each module looks like and what it costs.

| Card | Screenshot | Credit |
|------|-----------|--------|
| Vehicle overview | `assets/report/features/feat-overview.jpg` | 1 |
| MOT & mileage | `assets/report/features/feat-mot.jpg` | 3 |
| Digital service book | `assets/report/features/feat-dsb.jpg` | 3 |
| Grade-based valuation | `assets/report/features/feat-valuation.jpg` | 2 |
| Build sheet & spec | `assets/report/features/feat-buildsheet.jpg` | 2 |
| AI visual assessment | `assets/report/features/feat-ai.jpg` | 4 (paid add-on) |

- New card layout: screenshot (16:9, `object-fit:cover`, top-aligned) + credit badge overlay,
  then a small icon, title and description below. Removed the old big icon-chip.
- Images rendered at 130 DPI from `Sample_AnalyzeMyCar_Report_2026-05-22.pdf`, cropped to each
  section's content band, lazy-loaded with explicit width/height.
- Added a one-line intro: "Each card below is a real section from a sample report..."

### 2. "A look inside your report" gallery simplified
With every module now showing its own screenshot, the 3-image gallery (valuation + overview +
trend) was redundant. Reduced to a single featured **valuation page** with a caption. Overview and
trend now live in the feature cards. Removed the unused `report-overview.jpg` / `report-trend.jpg`
references from this section (the files remain in `assets/report/` and can be reused).

## QA method
Rendered the live file in headless Chromium (Playwright) at desktop (1280px) and mobile (390px);
screenshotted the features section and a single card to confirm the screenshot-led layout reads
well and the credit badge is legible at both widths.
