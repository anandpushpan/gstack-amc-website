# v2.3 Changes: Deep-dive compare blocks stack image-first on mobile

**Date**: 2026-06-13

## The bug

The three deep-dive sections ("Everything the competition is missing", "Service
and build records", "MOT trends and valuations") used inline-styled two-column
grids (image beside content) with no responsive class. They never collapsed on
mobile, so at phone widths the image and text sat cramped side by side.

## The fix

- Added a `split` class to all six two-column blocks.
- On mobile (<=900px) each `.split` now stacks to a single column with the
  **image above the content** (`figure` forced first, text below), matching the
  requested layout. Desktop keeps the side-by-side arrangement.

## Verification
- All six blocks confirmed single-column and image-first at 425px.
- Rendered and eyeballed: image on top, heading + copy + bullets below.
- Desktop unchanged. 0 em/en dashes.
